# OOP course - CS202 course

## "Strategy" design pattern

Tasks:

- A report in Markdown
- A presentation in PPTX/Google Slides/Canva
- A folder containing all source code
- A list of references

The report must contain:

1. A real-world problem (that needs the pattern to solve)
2. A naive solution (without the pattern)
3. Some problems that occur with the naive solution
4. An introduction about the pattern
5. A general class diagram
6. A class diagram for the problem in step 1
7. The implementation of the pattern
8. Pros and cons of the pattern
9. Some other real-world problems, how the design pattern is applied in web dev, mobile dev, etc.
10. Quiz: 5-8 questions, multiple choices
11. For each group presenting their first design pattern, there are 2–3 other groups working on the same pattern.

These other groups will act as reviewers for the presenting group.

Reviewers are responsible for:

- Listening carefully to the presentation.
- Asking questions for clarification.
- Challenging assumptions or design decisions.
- Facilitating discussion and deeper understanding of the pattern.
